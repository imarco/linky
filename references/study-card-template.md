# Knowledge Card Template

This template defines the Markdown structure for study mode knowledge cards.
The AI agent uses this as a guide when assembling the final output.

## Template

```markdown
# {title}

> Source: {source_url} | Type: {content_type} | Date: {date}
> Verified: {verified_count} claims | Tagged: [争议:{disputed}] [过时:{outdated}] [待确认:{pending}]

## Overview

{2-3 sentence summary of the content}

## Core Concepts

### {concept_1_name}

{definition, explanation, examples}

### {concept_2_name}

{definition, explanation, examples}

<!-- Add 3-7 concepts total -->

## Prerequisites

{What the reader should know before studying this material}

## Key Claims & Evidence

- **Claim**: {claim_text} — [{tag}] {evidence_summary}
  - Tag options: [已验证] [存在争议] [已过时] [待确认]

<!-- Add all significant claims -->

## Code Examples

```{language}
{code_block}
```

{annotation: what this code does, why it matters}

<!-- Add all relevant code examples -->

## Cognitive Map

```mermaid
{mermaid_diagram}
```

## FAQ

**Q1**: {question_1}
**A1**: {answer_1}

**Q2**: {question_2}
**A2**: {answer_2}

<!-- Generate 5-8 questions a learner would ask -->

## Critical Assessment

### Strengths
- {strength_1}
- {strength_2}

### Weaknesses
- {weakness_1}
- {weakness_2}

### What's Missing
- {gap_1}
- {gap_2}

## Further Reading

- [{resource_title}]({url})
- [{resource_title}]({url})
```

## Notes

- `{placeholder}` values are filled by the AI agent during study mode execution
- The cognitive map is generated by `scripts/linky/study.py:generate_cognitive_map`
- Truth anchoring tags are applied during the verification step
- Code examples should be annotated with context, not just pasted
